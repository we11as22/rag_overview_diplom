# Текущая ситуация по дипломной работе

**Тема:** обзор различных техник Retrieval-Augmented Generation (RAG).  
**Текущий реализованный кейс:** бот по документации на датасете Wix/WixQA.  
**Цель промежуточной сдачи:** показать, что есть рабочий прототип, экспериментальный контур и понятный план расширения диплома до обзора нескольких RAG-подходов.

## 1. Что уже сделано

Реализован базовый end-to-end сценарий RAG для вопросно-ответного бота по документации:

- загружен и подготовлен датасет Wix/WixQA;
- настроена фильтрация корпуса: из 6 221 документов в индекс попадает 4 172 статьи (`article` + `known_issue`, без `feature_request`);
- реализован retrieval-бенчмарк в `01_retrieval_eval`;
- сравнены BM25, dense vector search, hybrid linear fusion и RRF;
- сравнены embed-модели `embeddinggemma`, `qwen3-embedding:0.6b`, `bge-m3`;
- подобраны параметры retrieval для двух инструментов агента:
  - `search_by_chunks`: hybrid linear, `embeddinggemma`, `alpha=0.7`, `top_k=8`;
  - `search_by_titles`: vector, `embeddinggemma`, `alpha=1.0`, `top_k=10`;
- реализован RAG-сервис на FastAPI + PostgreSQL + pgvector;
- реализован ADK-агент с инструментами поиска, открытия статьи, workspace, memory и сжатием контекста;
- подготовлены отчёты по экспериментам и краткая презентация.

## 3. Текущие результаты экспериментов

### Поиск по чанкам

Лучший метод: **hybrid linear** с `embeddinggemma`, `alpha=0.7`.

| Метод | MRR@10 | Recall@10 |
|-------|--------|-----------|
| BM25 | 0.341 | 0.547 |
| Vector (`embeddinggemma`) | 0.543 | 0.872 |
| Hybrid linear (`alpha=0.7`) | **0.577** | 0.865 |
| Hybrid RRF (`w=0.7`) | 0.495 | 0.851 |

Вывод: для чанков dense retrieval даёт основной прирост относительно BM25, а linear hybrid дополнительно улучшает ранжирование.

### Поиск по заголовкам

Лучший метод: **vector search** по title с `embeddinggemma`.

| Метод | Hit@10 | MRR@20 | Hit@20 |
|-------|--------|--------|--------|
| BM25 title | 0.290 | 0.187 | 0.385 |
| Vector title | **0.743** | 0.469 | 0.818 |

Вывод: для коротких заголовков BM25 сильно уступает dense retrieval, потому что формулировка пользовательского вопроса часто не совпадает с названием статьи.
